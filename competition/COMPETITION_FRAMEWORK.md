# 🏆 Mark Six 每日预测大赛 (M615042026-B Competition)

> **Project Vision**: 找到"预测随机的一般性系统和方法论"  
> **Competition Objective**: 通过两个 AI Team 的竞争性预测，探索并验证不同预测方法的有效性  
> **Core Belief**: *"Never say 不存在可供预测随机"* — 保持开放，直到有充分证据为止

---

## 🎯 大赛机制

### 参赛团队

| Team | 初始策略方向 | 核心假设 |
|------|------------|---------|
| **Team Alpha** | 统计分析派 (频率分析/贝叶斯推断) | 历史出现频率可预测未来分布 |
| **Team Beta** | 机器学习派 (神经网络/时序模式) | 高维特征空间存在隐藏模式 |

### 评分规则

每轮预测后，根据与实际开奖结果的**主号匹配数**计算得分：

| 匹配主号数 | 得分 |
|-----------|------|
| 6个主号全中 | **+10 分** (Jackpot!) |
| 5个主号相同 | **+4 分** |
| 4个主号相同 | **+2 分** |
| 3个主号相同 | **+1 分** |
| ≤2个主号相同 | **-10 分** |

### 每轮流程

```
1. DATA PIPELINE  → 获取最新开奖结果 (或等待开奖)
2. TEAM ALPHA     → 基于历史数据 + 自己的方法论生成预测 (6个主号)
3. TEAM BETA      → 基于历史数据 + 自己的方法论生成预测 (6个主号)
4. ARBITER (Simon) → 公布实际结果，计算双方得分，更新 Leaderboard
5. 两队各自复盘   → 分析为何对/错，调整下轮策略
```

## 🔬 与项目 M615042026-B 的关系

本大赛是 **Phase 3** 的一部分，目标是：

1. **验证** Phase 2 随机系统分类框架中的各种方法
2. **探索** Koopman DMD / MLP / 信息论 等方法在实际预测中的表现
3. **对比** 统计分析 vs 机器学习 两条技术路线
4. **积累** 证据，判断 Mark Six 是否存在可供预测的信号

> **注**: 即使两个 Team 都接近随机得分，这个结果本身也是有价值的——它为项目提供了"方法论在随机系统上的基准表现"。
