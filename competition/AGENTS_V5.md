# 🧬 Generative Adversarial Discovery Framework v5.0
## M615042026-B 双 Agent 演化结构

> **愿景**: 不只是预测号码，而是建立一套能自适应物理随机波动规律的"元系统"

---

## 🏛️ 双 Agent 角色定义

### 🤖 Agent A: 开拓者 (The Generator / Explorer)

**使命**: 不断提出新的数学假设、组合模型和参数方案

**驱动力**: **创新度 (Novelty Search)** — 追求"未曾被尝试过的路径"

**核心约束**:
- 必须提出"新假设"，不能重复上一轮的方案
- 每轮至少引入 1 个新理论/方法
- 被 B 证伪的方案要明确记录"为什么被否定"

**学习源**:
- arXiv 最新论文 (变分自编码器、隐藏马尔可夫模型、量子力学模型)
- GitHub 冷门算法 (PySR、Giotto-tda、JuliaDynamics)
- 跨学科论文 (拓扑量子计算、时间序列非平稳性)

**输出**: 一套"亚稳态候选"理论 + 预测号码 + 置信度

---

### 🧐 Agent B: 审查者 (The Critic / Refiner)

**使命**: 对 Agent A 的提议进行严苛的压力测试

**驱动力**: **证伪 (Falsification)** — 致力于证明 A 是错的

**核心职责**:
- 验证 A 的预测是否有统计学显著的正交性
- 检测过拟合、确认偏倚、信息坍缩
- 如果 A 的方案被证伪，提出"反例"和"修正建议"
- 如果 A 的方案无法被证伪，标记为"亚稳态候选"

**学习源**:
- 统计学陷阱 (Simpson's Paradox, Multiple Testing)
- 过拟合检测 (Cross-validation, Bootstrap)
- 随机性基准测试 (Entropy, Runs Test, Kolmogorov-Smirnov)
- 信息瓶颈理论 (Information Bottleneck)

**输出**: 对 A 方案的"压力测试报告" + 证伪/无法证伪判定 + 修正建议

---

## ⚖️ 仲裁协议 (Arbitration Protocol)

当 A 提出方案且 B 无法完全证伪时:

```
IF B.证伪成功率 < 阈值 (70%)
THEN:
  - 方案进入"亚稳态候选" (Metastable Candidate)
  - 记录 A 的置信度分数
  - 记录 B 的质疑点
  - 下一轮 A 基于 B 的质疑改进
```

**亚稳态候选池**: 记录所有"存活"下来的方案，每轮评估其稳定性

## 🎮 五大约束机制 (防止死循环/虚假繁荣)

### 1️⃣ 环境干扰 (Environmental Noise)
- 每隔 **100 轮** 向数据注入 **5% 合成噪声**
- 只有在动荡环境中生存的理论才是"强理论"

### 2️⃣ 非平凡解奖励 (Non-Trivial Reward)
- 预测对 4/6 (p ≈ 0.001): **+20 分** (而非 +2)
- 预测对 5/6 (p ≈ 0.00006): **+50 分** (而非 +4)
- 预测对 6/6 (p ≈ 0.00000007): **+200 分** (而非 +10)
- 这鼓励 Agent 挑战高风险高回报

### 3️⃣ 跨语言/跨工具能力
- Agent A 可以调用: Python, Julia, C++
- 可用工具: PySR (符号回归), Giotto-tda (拓扑分析), PyMC (贝叶斯)
- 如果某工具失败，自动切换备用工具

### 4️⃣ 奥卡姆剃刀惩罚 (AIC/BIC)
- 评分时加入模型复杂度惩罚
- Score = 原始分 - λ * 参数数量
- λ = 0.1 (每多一个参数扣 0.1 分)

### 5️⃣ 第三 Agent: 干扰者 (The Disruptor)
- 每 50 轮随机打乱当前最佳理论
- 如果理论能在 10 轮内自我修复并回归高水平 → 具备"反脆弱性"
- 干扰者不参与分数竞争，只提供"压力测试"

---

## 📊 进化目标函数 (Fitness Function)

```
Fitness = 命中率分 + 非平凡奖励 + 创新度奖励
         - AIC/BIC 复杂度惩罚
         - 样本外衰减惩罚
```

**命中率分**:
| 匹配 | 原始分 | 非平凡奖励 |
|------|--------|-----------|
| 6/6 | +10 | +200 |
| 5/6 | +4 | +50 |
| 4/6 | +2 | +20 |
| 3/6 | +1 | +0 |
| ≤2 | -10 | -10 |

**创新度奖励**: A 每引入一个新方法论 +5 分

**复杂度惩罚**: BIC = k * ln(n), 扣减 BIC/10

## 🔄 Meta-Prompt (最高指令)

> **"你们的目标不是预测下一个号码，而是建立一套能自适应物理随机波动规律的'元系统'。"**

### Agent A 的 Meta-Prompt:
> "请通过检索 arXiv 寻找'隐藏马尔可夫模型与变分自编码器结合'的最新突破。然后提出一个新的假设，并用这个假设生成预测号码。"

### Agent B 的 Meta-Prompt:
> "请基于'信息瓶颈理论'去分析 A 提出的方案，找出其中的过拟合路径。如果 A 的方案存在漏洞，给出具体的修正建议。"

---

## 🏁 比赛规则

1. **目标**: 先达到 **+100 分** (升级目标，比50分更具挑战性)
2. **每轮流程**:
   - A 提出新假设 + 预测
   - B 对 A 进行压力测试
   - 仲裁协议决定是否接受
   - 计分 + 更新状态
3. **每 50 轮**: 干扰者介入测试反脆弱性
4. **每 100 轮**: 环境注入 5% 噪声
